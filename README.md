# 🛡️ Wazuh SIEM Home Lab — SIEM & File Integrity Monitoring

A hands-on cybersecurity home lab deploying **Wazuh** as a SIEM platform, with a Windows agent reporting to an Ubuntu-based Wazuh Manager, and real-time **File Integrity Monitoring (FIM)** configured via Syscheck.

---

## 📌 Overview

This lab demonstrates:
- Deploying a full Wazuh stack (Manager + Indexer + Dashboard) on Ubuntu Server in VirtualBox
- Enrolling a Windows host as a monitored Wazuh agent
- Configuring real-time File Integrity Monitoring (FIM) on a Windows directory using Syscheck
- Viewing and triaging alerts in the Wazuh Dashboard

**Wazuh capabilities covered:**
- Log analysis
- File integrity monitoring (FIM / Syscheck)
- Intrusion detection
- Real-time alerting

## ⚙️ Installation Steps

### 1. Install Wazuh Manager on Ubuntu

**Add the Wazuh GPG key:**
```bash
curl -s https://packages.wazuh.com/key/GPG-KEY-WAZUH | sudo gpg --dearmor -o \
  /usr/share/keyrings/wazuh-archive-keyring.gpg
```

**Download and run the all-in-one installation script:**
```bash
curl -sO https://packages.wazuh.com/4.12/wazuh-install.sh && sudo bash ./wazuh-install.sh -a -i
```

Flags:
- `-a` — installs all components (Manager, Indexer, Dashboard)
- `-i` — runs in interactive mode

The installer will display the dashboard credentials at the end of the script. **Save these immediately.**

**If the indexer needs manual installation:**
```bash
sudo apt -o Acquire::ForceIPv4=true install wazuh-indexer=4.12.0-* -y
```

---

### 2. Access the Wazuh Dashboard

Check the Ubuntu VM's IP:
```bash
ifconfig
```

Open a browser and navigate to:
```
https://<ubuntu-vm-ip>
```

- Accept the self-signed certificate browser warning
- Log in with the credentials generated by the install script
- Default username: `admin`

> ⚠️ **Security note:** Change the default admin password immediately after first login. Never commit credentials to a public repository — use a `.env` file or secrets manager.

---

### 3. Install the Wazuh Agent on Windows

1. Download the latest Wazuh Agent MSI from the [official Wazuh documentation](https://documentation.wazuh.com/current/installation-guide/wazuh-agent/wazuh-agent-package-windows.html)
2. Install the MSI using default settings
3. The Wazuh Agent Manager GUI will be available from the Start Menu

---

### 4. Register the Windows Agent with the Manager

**On the Ubuntu Manager — generate the agent key:**
```bash
sudo /var/ossec/bin/manage_agents
```

- Select `A` to add a new agent
- Assign a name (e.g., `WINDOWS-AGENT`)
- Leave IP address blank (or enter the Windows host IP if using static addressing)
- After creation, select `E` to extract the key
- Copy the full base64 key string

**On Windows — apply the key:**
1. Open **Wazuh Agent Manager** from the Start Menu
2. Paste the agent key into the key field
3. Enter the Ubuntu Manager IP address
4. Click **Save** and **Restart** the agent service

---

### 5. Verify Agent is Active

- Open the Wazuh Dashboard
- Navigate to **Agents**
- Confirm the Windows agent status shows **Active**

## 🔍 File Integrity Monitoring (FIM) Configuration

Wazuh uses **Syscheck** for real-time file and folder monitoring.

### Edit the Agent Configuration on Windows

Open the config file as Administrator:
```
C:\Program Files (x86)\ossec-agent\ossec.conf
```

Add the following inside the `<syscheck>` block:
```xml
<directories realtime="yes">C:\Users\<your-username>\Test</directories>
```

Replace `<your-username>` with the actual Windows username.

### Restart the Wazuh Agent

Restart via Services or the Agent Manager GUI to apply the configuration change.

---

## ✅ Verifying FIM Alerts

1. Open the **Wazuh Dashboard**
2. Navigate to **Agents** → select your Windows agent
3. Go to **Integrity Monitoring**
4. Perform file operations in the monitored folder:
   - Create a new file
   - Modify an existing file
   - Delete a file
5. Confirm that **FIM alerts appear** in the dashboard with event details (file path, hash, modification type)

## 🔐 Key Concepts Demonstrated

- **SIEM deployment** — full Wazuh stack installation using the all-in-one installer
- **Agent enrollment** — generating and applying agent keys to register endpoints with the manager
- **File Integrity Monitoring (FIM)** — real-time detection of file create/modify/delete events via Syscheck
- **Alert triage** — viewing and interpreting FIM alerts in the Wazuh Dashboard
- **Bridged networking** — placing the Ubuntu VM on the same subnet as the Windows host for agent communication
- **MITRE ATT&CK relevance** — FIM maps to T1565 (Data Manipulation) and T1083 (File and Directory Discovery) detection

## 🔒 Security Best Practices

- Change the default Wazuh admin password immediately after installation
- Never commit passwords, agent keys, or API tokens to a public repository
- Use a `.env` file or secrets manager for any credential storage
- Keep the Wazuh stack updated to receive the latest detection rules
- Restrict dashboard access to trusted IPs only in production environments
